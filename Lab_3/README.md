# Классификация временных рядов и их прогнозирование на примере метеоданных шести городов России

## Описание проекта

Целью проекта является разработка системы анализа временных рядов для автоматической классификации климатического типа и прогнозирования погодных параметров на основе исторических метеоданных. Решение охватывает полный цикл работы с временными рядами: от загрузки и предобработки данных до построения моделей классификации и прогнозирования температуры на месяц вперед.

## Используемые данные

Проект реализуется для регионального центра гидрометеорологии, осуществляющего мониторинг погодных условий на территории Российской Федерации. Центр использует данные метеостанций шести городов России:

- Москва
- Санкт-Петербург
- Сочи
- Геленджик
- Благовещенск
- Находка

Исходные данные получены из open-meteo и содержат почасовые или ежедневные наблюдения за период 2019–2025 годов. Для каждого города доступны следующие параметры:

- температура воздуха (`temperature_2m`)
- относительная влажность (`relative_humidity_2m`)
- осадки (`precipitation`)
- дождь (`rain`)
- снег (`snowfall`)
- погодный код (`weathercode`)
- скорость ветра (`wind_speed_10m`)
- атмосферное давление (`surface_pressure`)

## Основные этапы работы

### 1. Разведочный анализ данных (EDA)

На данном этапе для каждого города были выполнены:

- анализ всех показателей телеметрии: тренды, сезонность (суточная, годовая), выбросы, пропуски
- исследование распределения каждого признака (гистограммы, box-plot, Q-Q plot для проверки нормальности)
- выявление и визуализация сезонных паттернов (по месяцам, по временам года)
- анализ корреляции между всеми признаками (тепловая карта корреляционной матрицы)

Также был проведён кросс-городской анализ:

- сравнение распределения температуры и осадков между городами
- визуализация климатических профилей городов (среднемесячные температуры, среднегодовые осадки)
- выявление аномалий и выбросов в межгородском сравнении
- анализ пропусков

И выполнен анализ стационарности.

### 2. Feature Engineering

Для повышения качества моделей были сформированы дополнительные признаки:

- временные признаки (циклические признаки sin / cos для часа и месяца для учёта циклического характера времени)
- лаговые признаки
- скользящие статистики (mean, std, min, max)
- агрегированные временные характеристики процесса

### 3. Построение модели классификации

Был проведён анализ существующих решений для классификации временных рядов.

В качестве основной модели был выбран `RocketClassifier` из библиотеки sktime, так как данный алгоритм:

- специально разработан для классификации временных рядов;
- эффективно работает на многомерных временных рядах;
- показывает высокое качество на задачах временной классификации;
- обладает высокой вычислительной эффективностью.

Для обучения классификатора использовались окна фиксированной длины временного ряда, содержащие последовательности метеопараметров за выбранный период наблюдений.

### 4. Оценка качества классификации

Для оценки качества классификации использовались следующие метрики:

- Accuracy
- Precision
- Recall
- F1-score (macro)
- F1-score (weighted)

Дополнительно были построены:

- матрица ошибок (confusion matrix) для анализа типичных ошибок классификации
- ROC-AUC для многоклассовой схемы One-vs-Rest

В рамках дополнительного анализа были исследованы:

- наиболее трудноразличимые климатические зоны и города;
- влияние сезона на качество классификации;
- различия качества в зимний, летний и переходные периоды;
- динамика ошибок классификации по месяцам и годам.

Для интерпретации модели проводился анализ важности признаков с помощью permutation importance.

Также были визуализированы:

- распределения ошибок классификации;
- изменения качества модели во времени;
- confusion matrix;
- ROC-кривые для каждого класса.

### 5. Построение моделей прогнозирования

Для каждого города была построена отдельная модель прогнозирования температуры воздуха.

Формировалась выборка следующего вида:

- признаки — значения метеопараметров за предыдущие N дней;
- целевая переменная — температура воздуха на горизонте прогнозирования H = 30 дней.

Данные были разделены на:

- 2019–2023 — обучающая выборка;
- 2024 — валидационная выборка;
- 2025 — тестовая выборка.

Для прогнозирования исследовались различные модели временных рядов:

- SARIMAX (SARIMAX_EXOG)
- Exponential Smoothing (ETS)
- AutoReg

Для каждой климатической зоны подбиралась наиболее подходящая архитектура модели.

Было обосновано, что различные климатические условия требуют разных подходов:

- города с выраженной сезонностью лучше описываются сезонными моделями;
- регионы с высокой погодной изменчивостью требуют более гибких autoregressive-подходов;
- стабильные климатические зоны лучше моделируются сглаживающими моделями.

Дополнительно был проведён анализ моделей, плохо подходящих для месячного прогнозирования:

- простые линейные модели;
- модели без учёта сезонности;
- методы, не учитывающие временные зависимости;
- короткие autoregressive-модели без длинной памяти.

В процессе обучения:

- создавались лаговые признаки;
- вычислялись rolling statistics;
- использовались циклические календарные признаки;
- выполнялась проверка стационарности временных рядов;
- проводился автоматический выбор лучшей модели по качеству на validation-наборе.

Итоговая система строила прогноз температуры для каждого дня следующего месяца отдельно для каждого города.

### 6. Оценка качества прогнозирования

Для комплексной оценки качества прогнозирования использовались следующие метрики:

- **MAE** (Mean Absolute Error)
- **RMSE** (Root Mean Squared Error)
- **MAPE** (Mean Absolute Percentage Error)
- **WAPE** (Weighted Absolute Percentage Error)
- **Directional Accuracy** (точность предсказания направления изменений)

Был выполнен анализ остатков моделей:

- проверка нормальности распределения остатков (тест Шапиро–Уилка);
- анализ автокорреляции остатков (тест Льюнга–Бокса);
- проверка гетероскедастичности (тест Уайта).

Для визуального анализа результатов были построены:

- графики фактических и прогнозных значений на тестовом периоде;
- box-plot распределения ошибок для разных городов;
- графики прогноза температуры на месяц вперёд;
- визуализации распределения остатков моделей.

### 7. Интеграция этапов в единый пайплайн

В финальной части проекта этапы классификации климатического типа и прогнозирования температуры были объединены в единый двухэтапный пайплайн.

На первом этапе система получает последние 30 дней наблюдений температуры и определяет наиболее близкий климатический профиль среди известных городов. Для сравнения временных рядов использовалась метрика Dynamic Time Warping (DTW), позволяющая измерять сходство временных последовательностей с учетом возможных временных сдвигов и различий в динамике.

Для каждого входного временного ряда:

1. Формируется окно последних температурных наблюдений фиксированной длины;
2. Вычисляется DTW-расстояние до временных рядов всех городов;
3. Определяется наиболее похожий город (климатический аналог);
4. Выбирается лучшая модель прогнозирования, ранее обученная для данного города.

Такой подход позволяет учитывать климатические особенности регионов и автоматически подбирать наиболее подходящую модель прогнозирования.

На втором этапе используется соответствующая прогнозная модель, обученная для выбранного климатического типа. Система формирует прогноз температуры на следующие 30 дней.

В рамках эксперимента была реализована функция:

- классификации города по последнему температурному окну;
- автоматического выбора модели прогнозирования;
- генерации прогноза на месяц вперед.

Дополнительно выполнялась проверка работы пайплайна на всех городах выборки. Для каждого города:

- определялся наиболее близкий климатический аналог;
- вычислялось DTW-расстояние;
- выбиралась специализированная модель;
- строился прогноз температуры на ближайшие 30 дней.

Таким образом, реализованный пайплайн объединяет:

- классификацию временных рядов;
- выбор специализированной модели;
- прогнозирование погодных параметров;

в единую автоматизированную систему анализа метеоданных.

## Результаты

По результатам сравнительного анализа наилучшее качество прогнозирования показала модель **SARIMAX** (SARIMAX EXOG).

Преимущества данной модели:

- наименьшее значение ошибки прогнозирования среди рассмотренных моделей;
- хорошее воспроизведение сезонной и долгосрочной динамики температурных рядов;
- возможность учитывать внешние метеорологические признаки (exogenous features);
- устойчивость к сезонным колебаниям и трендам;
- приемлемое время обучения и построения прогноза.

Использование дополнительных экзогенных признаков, лаговых переменных и календарных факторов позволило повысить точность месячного прогнозирования температуры.

Дополнительно было установлено, что интеграция классификации временных рядов с последующим выбором специализированной модели прогнозирования повышает адаптивность системы к различным климатическим условиям и улучшает качество итоговых прогнозов.
